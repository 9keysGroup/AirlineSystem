# Airline Reservation System

This project is a Java-based airline reservation system with a local web interface.

The application:

- starts a local HTTP server
- serves a browser UI on `http://localhost:8080`
- lets you search routes, manage bookings, inspect the flight network, and run a performance benchmark

## Requirements

- Java JDK 8 or later installed
- Windows PowerShell

This project was tested locally with:

- `C:\Program Files\Java\jdk1.8.0_201\bin\javac.exe`
- `C:\Program Files\Java\jdk1.8.0_201\bin\java.exe`

## Project Structure

- `src/main/java/org/example/AirlineReservationSystem.java`
  Starts the application
- `src/main/java/org/example/AirlineWebServer.java`
  Serves the web UI and API endpoints
- `src/main/java/org/example/AirlineSystem.java`
  Contains the graph, booking queue, Dijkstra logic, and benchmark logic

## How to Compile

From the project root, run:

```powershell
& 'C:\Program Files\Java\jdk1.8.0_201\bin\javac.exe' -d build\tmp\manual-compile src\main\java\org\example\*.java
```

If your JDK is on `PATH`, you can also use:

```powershell
javac -d build\tmp\manual-compile src\main\java\org\example\*.java
```

## How to Run

After compiling, run:

```powershell
& 'C:\Program Files\Java\jdk1.8.0_201\bin\java.exe' -cp build\tmp\manual-compile org.example.AirlineReservationSystem
```

If `java` is on `PATH`, you can also use:

```powershell
java -cp build\tmp\manual-compile org.example.AirlineReservationSystem
```

## Open the Application

Once the server starts, open this URL in your browser:

```text
http://localhost:8080
```

## Railway Deployment

This repo is prepared for Railway deployment with:

- dynamic port binding through the `PORT` environment variable
- a `Dockerfile` for deterministic builds
- a runnable jar generated by Gradle

### Deploy Steps

1. Push this repository to GitHub.
2. Create a new Railway project.
3. Choose `Deploy from GitHub repo`.
4. Select this repository.
5. Railway will build the app from the included `Dockerfile`.
6. Once deployed, Railway will assign a public URL automatically.

No manual port configuration is required because the app reads the Railway `PORT` variable at startup.

## Troubleshooting

### Port 8080 already in use

If the app does not start, another process may already be using port `8080`.

You can stop the current process with:

```powershell
Get-NetTCPConnection -LocalPort 8080 -State Listen | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```

Then run the app again.

### Browser shows old UI

If the page looks outdated after changes:

- refresh the page
- if needed, do a hard refresh to clear cached JavaScript and CSS

### `javac` or `java` not found

Use the full JDK path shown above, or install/configure Java so `javac` and `java` are available on `PATH`.

## Notes

- The flight data is currently hardcoded in the Java source.
- The application stores data in memory only while the program is running.
- Closing the Java process stops the server.
